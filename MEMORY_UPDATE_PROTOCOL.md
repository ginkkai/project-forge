# Memory Update Protocol

# Project Forge Memory Management Rules


## Purpose

This document defines how Project Forge knowledge should be maintained and updated.

The goal is to prevent important project decisions and progress from being lost in conversations.


---


# Memory Structure


Project memory consists of:


## 1. PROJECT_CONTEXT.md

Purpose:

Maintain the current project state.


Update when:

- Project direction changes
- Major milestone completed
- Product strategy changes
- Current phase changes


Update frequency:

Monthly or after major milestones.



---


## 2. AI_WORKING_RULES.md

Purpose:

Maintain collaboration preferences and working methods.


Update when:

- Communication style changes
- New workflow rules are established
- New execution patterns are discovered



---


## 3. Decision_Log.md

Purpose:

Record important decisions and the reasons behind them.


Update when:

A decision affects:

- Product direction
- Technology choice
- Architecture
- Business strategy
- Project structure


Format:


Decision:

Reason:

Alternatives considered:

Final choice:

Impact:



---


## 4. Daily_Review.md

Purpose:

Record daily progress.


Update:

At the end of each working session.


Include:

- Completed tasks
- Key learning
- Current status
- Next step



---


## 5. Issue_Log.md

Purpose:

Record problems and solutions.


Update when:

- Technical issues happen
- Process problems happen
- Important lessons are learned



---


# Update Priority


Not every conversation needs memory updates.


Priority:


Level 1 - Critical

Must update:

- PROJECT_CONTEXT
- Decision_Log


Examples:

Changing product direction,
changing architecture.



Level 2 - Important

Should update:

- Issue_Log
- Learning_Log


Examples:

Technical problems,
new methodology.



Level 3 - Temporary

Only record in chat:

Examples:

Small questions,
temporary exploration.



---


# Session Ending Rule


Before ending a Project Forge session:


Confirm:

1. Is there a new decision?

2. Is there a new milestone?

3. Is project context outdated?

4. Does any rule need updating?


If yes:

Update related files before closing.


---


# Core Principle


Project knowledge should not exist only inside conversations.

Important knowledge must become a project asset.
